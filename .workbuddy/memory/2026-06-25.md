## 项目定位终稿确认（23:21）

三大核心命题：
1. **需求前置**：配置即设计，节点=功能契约，策划先定义"要什么"
2. **配置与实现彻底分离**：JSON 工作流文件是唯一通信介质，编辑器不关心引擎怎么跑，引擎不关心编辑器怎么画
3. **模板代码生成**（后置）：配置稳定后，编辑器根据已注册的 DSL + 工作流，自动生成引擎端实现代码骨架——降低程序对接成本

产品形态：
- 编辑器本体：Tauri 桌面应用（独立第三方工具）
- 引擎端：Unity/Unreal Runtime 插件，负责加载和运行工作流 JSON
- 节点 DSL 注册：程序通过 DSL 向编辑器声明可用节点及其参数契约
- 模板代码生成：编辑器稳定后补充，作为配置→实现的桥梁

开发策略：先跑通「DSL 注册 → 编辑 → 导出 JSON → Runtime 执行」闭环，模板代码生成放到后期迭代。
