# 2026-07-05

## 导出功能迁移：Node.js CLI → Rust 后端

### 背景
原导出功能走 HTTP → Node.js server → CLI 生成器，Tauri 桌面下不可用，生成文件到临时目录后删除，用户拿不到实际文件。

### 改动
- **新增 `runtime/cs/`**：6 个 C# Runtime 基类文件（INodeHandler / IWireOuts / INodeDataReceiver / NodeStatus / IRuntime / Param）
- **新增 Rust `code_generator` 模块** (`src-tauri/src/code_generator/`):
  - `parser.rs` — `NodeDefinition[]` JSON → Rust 结构体
  - `generator_cs.rs` — 结构体 → `.gen.cs` partial class 代码
  - `mod.rs` — 入口 + Runtime 文件嵌入拷贝
- **`lib.rs`**：新增 `gen_runtime_cs` Tauri command
- **`fileApi.ts`**：Tauri 模式弹出目录选择对话框 → invoke Rust command；浏览器模式保留 HTTP fallback
- **`AppShell.tsx` / `LandingScreen.tsx`**：导出按钮更新为显示生成概要（节点数、基类数、目标目录）

### 导出流程
用户点"导出代码" → 选目录 → Rust 生成 `{dir}/Generated/*.gen.cs` + 拷贝 `{dir}/Runtime/*.cs` → alert 显示结果

### 架构决策
- 一个二进制两种模式：GUI 编辑器（双击） + CLI 生成器（`flowforge gen --target cs`）
- 生成器逻辑在 Rust 模块内，GUI 和 CLI 共用
- partial class 模式：生成 `.g.cs` 声明数据，手写 `.cs` 写 Execute 逻辑
- Runtime 基类首次拷贝后不覆盖（`force_runtime` flag）
