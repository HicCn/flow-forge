## Bug 修复：拖拽节点到画布无法创建
- **根因**：`FlowCanvas.tsx` 缺少 `onDragOver` 和 `onDrop` 事件处理器
  - `NodeLibrary.tsx` 正确设置了 `draggable` + `onDragStart` → `dataTransfer.setData('application/node-type', def.type)`
  - 但 ReactFlow 组件没有 `onDragOver`（需要 `preventDefault()` 允许 drop）和 `onDrop`（读取 dataTransfer 并调用 `addNode`）
- **修复**：在 `FlowCanvas.tsx` 中新增两个 handler：
  - `onDragOver`：`preventDefault()` + 设置 `dropEffect = 'move'`
  - `onDrop`：读取 `application/node-type` → 查找 `nodeDefinitions` → `screenToFlowPosition` 转换坐标 → `addNode(def, position)`
- 同时补上了 `addNode` 从 store 的绑定（之前只用了 `getState()` 模式在右键菜单中）
- TypeScript 编译 + Vite build 均通过

## Bug 修复 #2：拖拽仍不生效 — onDragOver/onDrop 不在 ReactFlow props 中
- **二次根因**：`@xyflow/react` 的 `<ReactFlow>` 组件**不接受** `onDragOver`/`onDrop` 作为 props（源码中无这两个属性），传给它会被静默忽略
- **修复**：将 `onDragOver` 和 `onDrop` 从 `<ReactFlow>` 移到外层 wrapper `<div>` 上
- 浏览器原生 drag 事件会从 ReactFlow 内部 DOM bubble 到外层 div，drop 事件在 wrapper 上正确触发
- HMR 热更新已生效，刷新页面即可测试
- TypeScript 构建通过，Vite 热加载正常
