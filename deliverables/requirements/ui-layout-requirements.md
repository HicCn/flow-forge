# FlowForge 编辑器 UI 布局与交互需求

**日期**：2026-06-26
**版本**：v1.0
**依赖**：[产品方案 PRD](../product-strategy/prd-flow-editor-game-dev-2026-06-25.md)

---

## 1. 设计原则

| 原则 | 说明 |
|---|---|
| **画布优先** | 中央画布占用最大面积，是唯一不可折叠的区域 |
| **渐进可见** | 节点库和属性面板可按需折叠，策划可最大化画布专注编辑 |
| **瞬时搜索** | 主要创建方式为 Cmd+K/右键搜索，节点库为辅助浏览 |
| **所见即所配** | 属性面板字段由 DSL 驱动，程序定义什么、策划看到什么 |
| **零模式切换** | 编辑、预览、校验在同一视图中完成 |
| **声明式数据流** | 节点间执行顺序走连线（flow），参数流转走 Parameters 面板声明，画布只保留控制流 |

## 3. 区域一：工具栏

**位置**：顶部，固定高度 40px。
**状态**：始终可见，不可折叠。

### 3.1 功能清单

| 功能 | 触发方式 | 说明 |
|---|---|---|
| 文件标题 | 始终显示 | 格式：`FlowForge — filename.flow`，未保存时加 · |
| 撤销 | Ctrl+Z / 按钮 | 支持 50 步历史，包含节点增删、连线、参数修改 |
| 重做 | Ctrl+Y / 按钮 | 与撤销对称 |
| 缩放 | 按钮 / Ctrl+滚轮 | 显示当前百分比（10%~500%），点击可输入数值 |
| 导出 | 按钮 | 下拉菜单：Export .flow / Export .yarn / Export .lua |
| 保存 | Ctrl+S / 按钮 | 写入 .flow JSON 文件，首次弹出保存对话框 |
| 设置 | 齿轮按钮 | 打开设置面板（语言、主题、自动保存间隔） |

### 3.2 文件已修改指示

- 标题前显示圆点：绿色 = 已保存，橙色 = 未保存
- 关闭未保存文件时弹出确认对话框

## 4. 区域二：Parameters 面板

**位置**：悬浮于画布左上角，不占用布局空间。可拖拽移动位置。
**状态**：默认展开，可通过右上角箭头折叠为一小条。折叠后仅显示"Params"标签，悬停展开。

### 4.1 设计目的

节点间**数据流转不通过连线**，而是通过声明式参数引用。Parameters 面板是整个工作流的数据中枢：
- 生产方（Flow Input / 节点产出）向面板写入参数
- 消费方（节点属性）从面板中选择引用参数
- 画布上只保留 `flow` 类型的执行流连线

### 4.2 面板内容（垂直列表）

```
┌─ Parameters ─────────┬ [＋] [▼] ┐
│                      │           │
│  ● npc_name          │           │
│    string  "铁匠老王"  │           │
│                      │           │
│  ● quest_id          │           │
│    number  42         │           │
│                      │           │
│  ● mood              │           │
│    enum    angry      │           │
│                      │           │
│  ● speaker           │           │
│    string  "铁匠老王"  │           │
│                      │           │
└──────────────────────┴───────────┘
```

| 功能 | 说明 |
|---|---|
| 参数条目 | 每行一个参数：名称（粗体）+ 类型标签 + 当前值预览 |
| Add 按钮 | 新增 Flow Input 参数或表达式 |
| 折叠箭头 | 折叠后仅显示"Params"小标签，吸附在左上角 |
| 类型标签 | 缩写标签（s=string, n=number, f=float, b=boolean, e=enum） |
| 来源标记 | 鼠标悬停条目显示来源：Flow Input / Node:{节点名} / Expression |
| 拖拽 | 面板可拖拽移动到画布任意位置 |

### 4.3 参数三种来源

| 来源 | 创建方式 | 示例 |
|---|---|---|
| **Flow Input** | 手动 Add 添加，或通过 Flow Input 节点自动创建 | `npc_name`、`quest_id` |
| **节点产出** | 某节点运行时写入，由 DSL 的 `writes` 字段声明 | `speaker`（对话后更新）、`hp`（战斗后更新） |
| **表达式** | 基于已有参数计算得出 | `is_hard = difficulty >= 3` |

### 4.4 面板交互

- 面板浮于画布之上，不挤压节点库和属性面板的空间
- 可拖拽到画布任意位置，位置持久化到用户设置
- 折叠状态下显示为左上角小标签 "Params ⬤4"，数字为参数总数
- 默认最小宽度 140px，可拖拽边缘调整宽度

## 5. 区域三：节点库

**位置**：左侧面板，宽度 150px。
**状态**：默认展开，可通过分隔线拖拽或按钮折叠为 0px。折叠后面板上沿显示一条竖线，鼠标悬停展开。

### 5.1 功能清单

| 功能 | 说明 |
|---|---|
| 搜索框 | 顶部固定，输入即过滤，支持拼音首字母 |
| 分类标题 | 大写标签，不可点击，纯分组标识 |
| 节点卡片 | 每个节点显示：名称 + 颜色标签（对应 DSL category） |
| 拖放创建 | 从列表拖拽节点卡片到画布指定位置 |
| 双击创建 | 双击卡片，节点出现在画布中心 |
| 最近使用 | 展开时底部常驻 3 个最近添加的节点 |
| 全部折叠/展开 | 分类标题右侧箭头，点击折叠该分类 |

### 5.2 分类示例

```
DIALOGUE          ▼
  Dialogue Line   🔴
  Branch Choice   🔴
  Condition       🔴
  Variable Set    🔴

FLOW CONTROL      ▼
  Loop            🔵
  Wait            🔵
  Parallel        🔵

EVENTS            ▼
  Send Event      🟢
  Trigger Cutscene🟢
```

### 5.3 复用节点库入口

节点库底部常驻一个 **Template** 分区，列出策划创建的复用节点组（Subgraph），交互同上。

## 6. 区域四：画布

**位置**：中央区域，宽度自适应。
**状态**：始终可见，不可折叠。空画布时显示引导文字。

### 6.1 画布自身

| 能力 | 说明 |
|---|---|
| 无限平移 | 鼠标中键拖拽 或 空格+左键拖拽 |
| 缩放 | 滚轮，以鼠标位置为锚点缩放，10%~500% |
| 背景网格 | 30px 间距浅灰网格线 |
| 居中适配 | 按 F 键，所有节点自动适配到视口 |
| 空画布引导 | 显示文字提示："按 / 或右键搜索添加节点" |

### 6.2 节点交互

| 操作 | 行为 |
|---|---|
| 单击选中 | 高亮边框，属性面板联动刷新 |
| 框选 | 拖拽矩形框，选中的节点显示集体高亮 |
| 拖拽移动 | 单个/多个同时移动 |
| Delete | 删除选中节点和所有连线 |
| Ctrl+C / Ctrl+V | 复制粘贴节点（保持相对位置，重新生成 UUID） |
| Ctrl+D | 原地复制 |
| 右键节点 | 上下文菜单：复制 / 删除 / 禁用 / 编组 / 重置参数 |

### 6.3 连线交互（仅 flow 类型）

| 操作 | 行为 |
|---|---|
| 拖出连线 | 从输出 flow Pin 拖出临时线 |
| 接入目标 | 拖到目标输入 flow Pin 上松手，无效目标显示禁止图标 |
| 连线颜色 | flow 连线统一为灰色 |
| 删除连线 | 单击选中连线，按 Delete |
| 断开重连 | 拖拽连线端点，重新吸附 |

> 注：画布上**仅显示 flow 类型 Pin 和连线**。数据流转通过 Parameters 面板声明式完成，不在画布上连线。

### 6.4 画布右键菜单

右键空白区域弹出搜索面板（同 Cmd+K），选中后节点出现在点击位置。

## 7. 区域五：属性面板

**位置**：右侧面板，宽度 180px。
**状态**：默认展开，可折叠。无选中节点时显示"选中节点以编辑属性"。

### 7.1 面板内容

当选中的是单个节点时，面板分三部分：

| 区域 | 说明 |
|---|---|
| 节点头部 | 节点名称 + 类型标签（颜色同分类） |
| 输入参数 | DSL 定义的 params 数组，动态渲染表单控件 |
| 节点产出 | DSL 定义的 writes 数组，列出该节点会写入的 Parameters |
| 元信息 | 节点 ID（只读）、添加时间（只读） |

### 7.2 参数来源切换

每个声明了 `source: "param_or_fixed"` 的参数，在属性面板中显示为双模控件：

```
  NPC Name
  ┌─────────────────────────────┐
  │ ○ 固定值                    │    ← 选中：显示输入框
  │   ┌─────────────────────┐   │
  │   │ 铁匠老王              │   │
  │   └─────────────────────┘   │
  │ ● 使用参数                  │    ← 选中：显示下拉选择
  │   ┌─────────────────────┐   │
  │   │ npc_name        ▼   │   │
  │   └─────────────────────┘   │
  └─────────────────────────────┘
```

| 控件 | 行为 |
|---|---|
| 固定值 | 显示对应类型的输入控件（文本/数字/下拉/开关） |
| 使用参数 | 下拉列出 Parameters 面板中**类型兼容**的所有参数 |
| 切换 | 点击单选框切换，值保留不动 |
| 被连线接管（灰色） | 当参数由 `expose_pin` 暴露且 Pin 已连线时，两个选项均禁用 |

### 7.3 节点产出（writes）显示

当节点 DSL 定义了 `writes` 数组时，属性面板底部额外显示产出信息：

```
  节点产出
  ┌─────────────────────────────┐
  │ speaker    → 写入 Parameters │
  │ hp         → 写入 Parameters │
  └─────────────────────────────┘
```

只读，不可编辑。

### 7.4 表单控件映射

| DSL 类型 | 控件 | 校验 |
|---|---|---|
| string | 单行文本输入 | 非空校验 |
| number | 数字输入框 | 范围 min/max |
| float | 数字输入框（小数步进） | 精度校验 |
| boolean | 开关 / 勾选框 | — |
| enum | 下拉选择 | 值必须在 options 内 |
| text | 多行文本区域（Markdown） | — |
| node_ref | 下拉选择（列出画布所有节点 ID） | 节点存在性 |
| asset | 资源选择器（文件路径 + 预览缩略图） | 文件存在性 |

### 7.5 多选模式

框选多个节点时，属性面板显示：
- 选中数量
- 共用可编辑参数（如"所有 Dialogue 节点的 speaker"）
- 批量操作按钮：编组 / 对齐 / 删除

## 10. 浮层：搜索面板

|---|---|

## 11. Pin 系统

画布上的 Pin **仅用于控制流（flow 类型）**。数据流转通过 Parameters 面板声明式完成。

### 11.1 画布可见 Pin

| 类型 | 颜色 | 说明 |
|---|---|---|
| flow | #888780 灰 | 执行流/控制流，决定节点执行顺序 |

节点上只显示 flow 类型的 Pin，外观为：
- **输入 Pin**：左侧，圆形实心，灰色
- **输出 Pin**：右侧，圆形空心，灰色
- **多 Pin 节点**：同侧 Pin 纵向排列，间距 18px

### 11.2 类型系统（面板引用用）

以下类型系统用于 Parameters 面板和属性面板的参数引用——在面板中通过颜色标签体现，不在画布上生成 Pin：

| 类型 | 颜色标签 | 用途 |
|---|---|---|
| flow | 灰 | 控制流，仅画布 Pin |
| string | 橙 | 文本 |
| number | 蓝 | 整数 |
| float | 蓝 | 浮点数 |
| boolean | 青 | 布尔值 |
| enum | 青 | 枚举值 |
| event | 绿 | 事件引用 |
| any | 紫 | 任意类型 |

### 11.3 属性面板中的引用校验

当用户在属性面板中将参数来源切换为"使用参数"时，下拉列表只显示**类型兼容**的 Parameters：

```
  参数来源：使用参数
  ┌─────────────────────┐
  │ quest_id     (number)│  ✅ 类型匹配
  │ npc_name     (string)│  ✅ string → string
  │ mood         (enum)  │  ✅ enum → string 兼容
  │ ─ ─ ─ ─ ─ ─ ─ ─ ─ │
  │ hp           (number)│  ❌ number → string 不兼容（置灰）
  └─────────────────────┘
```

兼容规则同 11.2 的类型兼容表：string → string；number → number/float；enum → string/enum；any → 全部。

### 11.4 可选 Pin

DSL 定义中 `required: false` 的 flow Pin 显示为虚线边框。不连线不影响校验通过。

## 14. 空状态与错误状态

| 场景 | 表现 |
|---|---|
| 空画布 | 网格背景 + 引导文字："按 / 搜索添加第一个节点" |
| 无选中节点 | 属性面板显示："选中节点以编辑属性" |
| 节点库无匹配 | 搜索框下方显示："无匹配节点" + 建议注册新节点 |
| 导出失败 | Toast 通知（顶部居中滑入），3 秒后消失 |
| 连线类型不匹配 | 松手后连线不建立 + Pin 短暂红色闪烁 |
| 文件未保存关闭 | 弹出确认框："Save changes to file.flow?" |
| 删除 DSL 定义被阻止 | 弹窗列出受影响的 .flow 文件及其中的实例数；需先替换或移除这些节点后方可删除 |

### 14.1 DSL 删除确认对话框

当用户尝试删除一个已在 .flow 文件中使用的节点定义时，显示阻断对话框：

```
┌─────────────────────────────────────────────┐
│  ⚠ 无法删除 "dialogue_line"                 │
│                                             │
│  以下文件中有 12 个节点实例正在使用此定义：    │
│                                             │
│  📄 chapter_1_intro.flow         4 个实例    │
│  📄 chapter_2_village.flow       8 个实例    │
│                                             │
│  请先将这些节点替换为其他类型或移除，          │
│  然后再尝试删除。                            │
│                                             │
│              [查看受影响的节点]   [取消]       │
└─────────────────────────────────────────────┘
```

点击"查看受影响的节点" → 在新 Tab 中打开对应 .flow 文件，定位到相关节点。

## 15. 响应式适配

编辑器最小窗口尺寸：**1024 × 768**。低于此尺寸时两侧面板自动折叠，画布全屏，通过键盘快捷键和搜索面板完成所有操作。
